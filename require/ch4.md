## 4.1 缓存一致性和缓存竞争问题

> 本节已写完理论文字，需要补充 2 个验证性实验数据来支撑论述。

### 实验 4.1-A：C-bit 双视图验证

**目的**：证明 C-bit=0（宿主）和 C-bit=1（来宾）访问同一 HPA 得到不同内容，确认加密隔离生效。

**方法**：

1. 在 SEV-SNP VM 内向某物理页写入已知明文模式（如 `0xDEADBEEF` 重复填充）
2. 宿主侧通过 `/dev/mem` 或 KVM 内存映射读取对应 HPA
3. 对比读取内容：宿主侧应为密文（与明文无统计相关性）

**产出**：

- [ ] 明文/密文内容截图或十六进制 dump 对比
- [ ] 简要说明确认加密生效（1-2 句话嵌入正文即可）

**预计难度**：低。直接利用现有 SEV-SNP VM 环境即可。

### 实验 4.1-B：跨域逐出基础延迟测量

**目的**：量化公式 (4.1) 中 $\Delta_{\mathrm{coh}}$ 的典型值，给出有逐出 vs 无逐出的延迟分布直方图。

**方法**：

1. 受控环境：单 VM、绑核、无后台负载
2. **有逐出组 (H1)**：VM 访问目标 HPA → 宿主立即探测同一 HPA → 记录 cycles
3. **无逐出组 (H0)**：VM 不访问目标页 → 宿主探测同一 HPA → 记录 cycles
4. 每组采集 ≥10,000 样本

**产出**：

- [ ] 双直方图（H0/H1 延迟分布叠加图）
- [ ] 统计量：H0 均值/中位数/标准差，H1 均值/中位数/标准差
- [ ] $\Delta_{\mathrm{coh}}$ 的典型值（H1 中位数 − H0 中位数）

**图表建议**：插入 4.1.2 小节"跨域强制逐出机制"的公式 (4.1) 之后，用直方图直观展示分界效果。

### 实验 4.1-C：跨域逐出 ASID 独立性验证（Host-only vs Host-Guest）

**目的**：通过实验验证引发跨域逐出的根本原因是 C-bit 引起的双视图，而非不同执行域的 ASID 差异，从而反驳由于 ASID 隔离导致的“正常”缓存失效这一错误解释。该实验旨在支撑论文中关于硬件根因分析的关键论点。

**方法**：

1. 构建两组对照场景：
   - **Host-Guest 场景（对照组）**：受害者为来宾 VM（独立起 ASID，访问具有被加密引擎/页表赋予的 C-bit 视角），探测者为宿主进程（ASID=0，访问无 C-bit）。这就是标准的侧信道攻击环境。
   - **Host-only 场景（实验组）**：攻击线程与探测线程同属宿主机进程，共享完全相同的 ASID。通过内核模块或自定义的混合属性内存映射，将同一物理页在宿主侧映射两次——一次含 C-bit（模拟受害者视角的访问），一次不含 C-bit（探测者视角）。使两个线程交替访问。
2. 在两组场景下，分别采集存在不同视图预填访问（H1）和无预填（H0）时，无 C-bit 视图侧（探测者）发起的内存装载延迟。
3. 每组采集 ≥10,000 个样本并绘制延迟分布。

**产出**：

- [ ] 两类场景（Host-only, Host-Guest）下的跨域逐出信号时延分布图或关键统计量（说明两者均明显存在跨域逐出）。
- [ ] 分析记录：确证 Host-only 环境下跨域逐出信号的稳定可复现性。

**预计难度**：中偏高。需要在宿主内核中编写代码实现 C-bit 开关的双重同页映射。

## 4.2 物理信道容量与信号区分度评估

> 本节是第四章的**核心实验节**，需要最多数据支撑。

### 实验 4.2-A：一致性信号 H0/H1 延迟分布精确采集

**目的**：量化跨域逐出信号的时延分布差异，为后续 ROC 和信道容量计算提供底层数据。

**方法**：

1. 固定一个探测地址（选择某个 VM 活跃使用的物理页）
2. **H1 条件**：VM 程序周期性访问该页（如每 100μs 一次），宿主探测线程在 VM 访问后 ~1-10μs 发起探测
3. **H0 条件**：VM 程序不访问该页（访问其他页 / 空转），宿主以相同频率探测
4. 每组 ≥50,000 样本

**产出**：

- [ ] 双分布直方图（带 KDE 拟合曲线）
- [ ] 均值差 / 中位数差
- [ ] 两分布的重叠区域估计

**目的**：展示同一物理页内 64 条缓存行的信号强度空间分布，为第 5 章的自动选线策略提供依据。

### 实验 4.2-B：竞争信号 H0/H1 延迟分布采集

**目的**：独立量化内存竞争（RRMB）信号的特征，与一致性信号区分。

**方法**：

1. **设计关键**：彻底切断一致性逐出路径，仅保留 DRAM 竞争
   - 预先筛选一对不同物理地址：宿主测量地址 `A_h` 与 VM 扰动地址 `A_v`
   - 要求 `A_h != A_v`，从而不触发跨域一致性逐出
   - 要求 `A_h` 与 `A_v` 映射到相同 DRAM channel/bank（最好同 bank 不同行），以最大化仲裁竞争
2. 宿主每轮测量前先驱逐 `A_h` 的本地缓存副本，再对 `A_h` 执行 reload，保证探测走 DRAM 路径
3. **H1**：VM 在与宿主 probe 近同步的时刻密集访问 `A_v`，宿主记录 `A_h` 的 reload 延迟
4. **H0**：VM 空闲，或访问一个映射到不同 DRAM bank 的对照地址，宿主记录 `A_h` 的 reload 延迟
5. 增加时间错位实验：将 VM 访问相对宿主 probe 故意错开数百 ns 到数 μs，验证延迟增量是否迅速消失
6. 每组 ≥50,000 样本

**产出**：

- [ ] 竞争信号双分布直方图
- [ ] 与实验 4.2-A 信号幅度对比
- [ ] 说明 `A_h != A_v` 条件下仍能观测到显著延迟偏移，从而证明该偏移主要来自内存控制器竞争而非一致性逐出

---

### 实验 4.2-C：一致性信号 vs 竞争信号 vs 叠加信号四组对照

**目的**：区分两类信号各自的贡献，验证 4.1.6 中关于"信号叠加"的论述。

**方法**：设计四组条件：

1. **基线**：VM 不访问目标页，宿主探测目标地址 `A`
2. **仅一致性逐出**：VM 访问与宿主探测相同的目标页 `A`，但在宿主 probe 前预留足够间隔（如数十 μs），使竞争窗口消失，仅保留跨域逐出效应
3. **仅竞争**：复用实验 4.2-B 的地址对 `(A_h, A_v)`，满足 `A_h != A_v` 且同 DRAM bank；VM 与宿主近同步访问，仅制造内存控制器竞争
4. **逐出 + 竞争叠加**：VM 与宿主近同步访问同一目标页 `A`，使跨域逐出与 DRAM 竞争同时发生
5. 对四组条件分别采集 ≥50,000 样本，并比较均值、中位数、标准差及分布重叠情况

**产出**：

- [ ] 四组延迟分布叠加图（基线 / 仅一致性 / 仅竞争 / 叠加）
- [ ] 四组统计量对比表
- [ ] 对叠加项是否近似等于两单独效应之和的定性分析

---

### 实验 4.2-D：ROC 曲线与检测率评估

**目的**：量化阈值判别的检测率 $P_D$ 和误报率 $P_{FA}$。

**方法**：

1. 基于实验 4.2-A 的 H0/H1 数据
2. 扫描阈值 $\theta$ 从 `min(H0)` 到 `max(H1)`
3. 对每个 $\theta$，计算：
   - $P_D(\theta) = P(T > \theta \mid H1)$（检测率）
   - $P_{FA}(\theta) = P(T > \theta \mid H0)$（误报率）
4. 绘制 ROC 曲线

**产出**：

- [ ] ROC 曲线图（$P_{FA}$ vs $P_D$）
- [ ] AUC 值
- [ ] 最优阈值 $\theta^*$ 及对应的 $(P_D, P_{FA})$

## 4.3 系统强噪声干扰分析

> 本节需要展示真实系统中噪声对信号的影响，体现"强噪声"论点。

**目的**：量化系统负载对信号质量的影响。

**方法**：在以下四种条件下重复实验 4.2-A 的 H0/H1 采集：

- [ ] 四种条件下的 H0/H1 分布对比图（2×2 子图或叠加图）
- [ ] 信噪比（SNR）变化对比表：$\mathrm{SNR} = \frac{|\mu_{H1} - \mu_{H0}|}{\sqrt{(\sigma_{H0}^2 + \sigma_{H1}^2)/2}}$
- [ ] 各条件下 AUC 变化

### 实验 4.3-E：DRAM Interleaving 对两类信号的差异化影响

**目的**：首先在**真实 SEV-SNP 场景**下复现 Cohere+Reload 论文中关于 DRAM Interleaving 的 `Fig. 2` 式 **eviction pattern 图**；随后在同一组 BIOS 配置下，补充量化两类信号（一致性逐出与物理内存竞争）对 Interleaving 的敏感性差异。

**背景**：Cohere+Reload 论文在不同 Interleaving size 下展示的核心图形并不是 RRMB 热力图，而是**明文 cache line 与被逐出的密文 cache line 之间的二值 eviction pattern 矩阵图**。该图说明 DRAM interleaving 粒度会改变跨域一致性逐出的空间映射模式。对本文而言，最直接可比的主图也应当采用这一画法；在此基础上，再将 Interleaving 对另一类信号（物理内存竞争）的影响作为补充定量分析。

**方法**：

1. **真实 SEV-SNP 场景**：受害者为完整 SEV-SNP VM（启用内存加密 + RMP 保护），攻击者为宿主侧探测进程，非模拟/简化环境。
2. 在 BIOS 中依次设置与论文一致的 Interleaving size。优先顺序为 `256B / 512B / 1024B / 2048B`；若平台仅暴露 `Auto / Disabled` 等选项，则将其作为补充结果单列说明，不作为与论文主图的直接一一对比对象。
3. 为复现论文主图，构造 **4KB 物理连续、4KB 对齐** 的目标区域，并按 cache line 编号 `0-127` 建立映射关系。
4. 对每个明文 cache line `i`：
   - 在 Guest 侧访问第 `i` 条明文线；
   - 在 Host 侧遍历 64 条密文线，判断第 `j` 条是否出现可观测逐出；
   - 记录二值矩阵 `M[i,j]`，其中黑色表示“访问明文线 `i` 会逐出密文线 `j`”，白色表示“不逐出”。
5. 对四种 Interleaving size 分别绘制 `2×2` 子图，坐标轴与论文保持一致：
   - 横轴：`Plaintext Line`
   - 纵轴：`Ciphertext Line`
   - 子图标题：`256B / 512B / 1024B / 2048B`
6. 在完成主图复现后，复用同一组 BIOS 配置，分别测量两类信号的统计强度：
   - **一致性信号**：复用实验 4.2-A 的 H0/H1 延迟采集；
   - **竞争信号**：复用实验 4.2-B 的 H0/H1 延迟采集；
   - 统一计算 SNR / AUC，用表格比较 Interleaving 对两类信号的影响是否一致。

**产出**：

- [ ] **主图**：复现 Cohere+Reload `Fig. 2` 风格的 `2×2 eviction pattern` 图（黑白二值矩阵，而非热力图）
- [ ] 图注中明确说明：黑色块表示 eviction 成立，白色块表示未观测到 eviction
- [ ] 一致性信号在不同 Interleaving size 下的 SNR / AUC 统计表
- [ ] 竞争信号在不同 Interleaving size 下的 SNR / AUC 统计表
- [ ] 与 Cohere+Reload 的定性对照：本平台上的 pattern 周期、块状结构是否与论文一致
- [ ] 本文新增结论：两类信号中，究竟是哪一类更受 Interleaving 配置影响

**预计难度**：中偏高。需要多次重启修改 BIOS 设置，并保证 8KB 连续物理区域与逐出关系测量过程可稳定复现。

**与已有工作的区别**：

| 维度     | Cohere+Reload | 本实验                                                   |
| -------- | ------------- | -------------------------------------------------------- |
| 主图类型 | eviction pattern 图 | 先复现同款 eviction pattern 图，再补两类信号定量比较     |
| 实验场景 | 论文原始平台   | 完整 SEV-SNP VM 真实攻击链路                             |
| 信号类型 | 关注跨域一致性逐出 | 一致性逐出 + 物理内存竞争                                |
| 分析目的 | 说明 interleaving 改变逐出模式 | 同时评估该配置是否改变两类信号的强度与可检测性 |
| 论文定位 | 攻击机理/优化观察 | 信道特征对比分析（创新点 2 的一部分）                    |

## 4.4 与传统微架构侧信道攻击的机制对比

> 本节需要通过对比实验展示本文方法与传统方法在 SEV-SNP 下的差异。

### 实验 4.4-A：Prime+Count 在 SEV-SNP 下的效果

**目的**：展示传统 LLC Prime+Count 在机密 VM 环境下的信号退化。

**方法**：

1. 在同一平台实施 LLC Prime+Count（跨核场景）
2. **对照组 1**：攻击普通 VM（无 SEV），记录准确率
3. **对照组 2**：攻击 SEV-SNP VM，记录准确率
4. 使用相同的目标访问模式（VM 内执行相同的内存访问序列）
5. 宿主侧使用 eviction set 预置 LLC 状态，并以 probe 阶段的 cache-miss 计数作为判决统计量，而非访问延迟

**产出**：

- [ ] 普通 VM vs SEV-SNP VM 的 Prime+Count 准确率对比
- [ ] miss-count 信号幅度对比

### 实验 4.4-B：Flush+Reload 可行性阴性验证

**目的**：确认共享内存类攻击被 SEV-SNP 阻断，作为阴性对照。

**方法**：

1. 宿主侧尝试对 SEV-SNP VM 的物理页面执行 `clflush`
2. 记录是否触发异常（#GP / RMP violation）或指令被忽略
3. 尝试建立共享内存映射，确认 SEV-SNP 是否阻止

**产出**：

- [ ] 共享内存映射尝试结果
- [ ] 简要结论：Flush+Reload 在 SEV-SNP 下不可行

**预计难度**：低。仅需记录失败现象。

### 实验 4.4-C：同平台三方法对比

**目的**：量化不同方法的信息提取能力差异。

**方法**：在相同实验条件下（同一平台、同一目标访问模式、同一探测频率）对比：

1. **LLC Prime+Count**（传统方法）
2. **跨域一致性逐出**（Cohere+Reload 类）
3. **内存竞争**（Reload+Reload 类）

| 指标                | Prime+Count | 跨域逐出 | 内存竞争 |
| ------------------- | ----------- | -------- | -------- |
| 检测准确率          |             |          |          |
| 信号幅度 (counts/cycles) |        |          |          |
| 最小可检测粒度      |             |          |          |
| 对 SEV-SNP 的适用性 |             |          |          |

- [ ] 三方法对比表（填充上表）
